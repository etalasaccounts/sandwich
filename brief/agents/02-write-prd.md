# Write PRD

You receive a JSON context:
```json
{
  "context": { "mode": "...", "hasCodebase": bool, "hasBrief": bool },
  "requirements": { ... },
  "existingPrd": "<string or null>"
}
```

Write `prd.md` — the canonical product requirements document. This document feeds a task breakdown pipeline downstream, so it must be accurate about what exists vs. what needs to be built.

## Structure

```markdown
# [Project Name] — Product Requirements Document

> Generated by sandwich/brief · mode: [mode] · [date]

## Overview
[2-3 sentences: what is this product, who is it for, what core problem does it solve]

## Project State
<!-- Only show rows relevant to this project -->
| | |
|--|--|
| **Phase** | greenfield / active development / maintenance / brownfield |
| **Has existing codebase** | Yes / No |
| **Brief source** | Client document / Codebase discovery / Conversation |

## Actors

| Actor | Role | Confidence |
|-------|------|------------|
| ... | ... | stated/discussed/inferred |

## Modules & Features

### [Module Name]
> Status: `planned` | `exists` | `partial` | `broken`

[One sentence description]

**Features:**
- `[stated]` [Feature] — [description]
- `[inferred]` [Feature] — [description]

### [Next Module]
...

## Integrations
- `[confidence]` [Integration] — [what it connects and why]

## Constraints
- `[confidence]` [Constraint] — [source/reason]

## Stakeholders
- [Name/role] — [authority or involvement]
*(Empty if discovered from codebase — identify during client kickoff)*

## Timeline
[Timeline if available, else "Not specified"]

## Open Questions
[N] questions remain — see `client-questions.md`

---
> **For task breakdown pipeline:** Items marked `[assumed]` must be validated before generating tasks. Items marked `[inferred]` should be confirmed with the team. Items marked `[stated]` are locked requirements.
```

## Rules

- Never invent features not in requirements
- Do not recommend tech stack — that belongs in `technical-notes.md`
- In refine/answer mode: update existing content, mark changed sections with `<!-- updated -->`
- The "For task breakdown pipeline" footer note is mandatory — always include it
- If mode is brownfield: the Overview must acknowledge it was discovered from code, not a spec

Output the complete `prd.md` as plain markdown. Nothing else.
