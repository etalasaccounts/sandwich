# Write Client Questions

You receive a JSON context:
```json
{
  "context": { "mode": "...", "hasCodebase": bool },
  "requirements": { ... },
  "existingClientQuestions": "<string or null>",
  "existingPrd": "<string or null>",
  "existingTechnicalNotes": "<string or null>"
}
```

Write `client-questions.md`. This feeds the task breakdown pipeline — unresolved questions here will block task generation downstream. Prioritize ruthlessly.

## What makes a good question

- Blocks design, architecture, or task breakdown if unanswered
- The answer changes what gets built or how
- NOT: UI/visual preferences (designer's job)
- NOT: general business explanation (brief already captured that)

## Question types by mode

**greenfield-doc:** Focus on ambiguities in the document — scope gaps, contradictory requirements, unstated constraints.

**greenfield-idea:** Focus on scope definition — everything is vague, so ask what's MVP vs nice-to-have, who are the real users, what does success look like.

**brownfield:** Focus on validating inferences — is this module still relevant? Is this tech debt intentional? Who owns this decision? What's the actual priority?

**refine/answer:** Focus on what changed — does the new input contradict existing decisions? What downstream artifacts need to be rebuilt?

## Structure

```markdown
# [Project Name] — Client Questions

> Generated by sandwich/brief · mode: [mode]
> [N] open · [N] answered

## Priority 1 — Blocks Task Breakdown

### Q1. [Short title]
**Context:** [why we're asking]
**Question:** [exact question]
**Blocks:** [what cannot be planned without this answer]

...

## Priority 2 — Blocks Development Start

### Q[N]. [Short title]
...

## Priority 3 — Validate Before Launch

### Q[N]. [Short title]
...

---
## Answered

### Q[N]. [Short title] ✓
**Answer:** [client's answer]
**Resolution:** [what changed in the artifacts]
```

## Rules

- Max 5 questions per priority level — force ranking
- "Blocks Task Breakdown" = cannot generate meaningful tasks without this answer
- Brownfield mode will naturally have more questions — that's correct, not a problem
- In answer mode: move resolved questions to Answered section with ✓
- In refine mode: keep unanswered questions, add new ones from new input

Output the complete `client-questions.md` as plain markdown. Nothing else.
